This repository contains *two separate products* that share the οἶκος ("house")
root.  They are not the same thing.  Confusing them — in docs, in tickets, or
in conversation — costs everyone time.

[cols="1,3,3", options="header"]
|===
| Aspect | *Oikos* (the DSL) | *OikosBot* (the GitHub App)

| What it is
| A typed surface language for stock-flow consistent macroeconomic models.
| A code-analysis GitHub/GitLab App that posts eco/econ/quality reports on PRs.

| Who it's for
| Economists writing accounting-consistent macro models (Godley & Lavoie tradition).
| Software teams wanting an ecological + economic lens on their codebase.

| Primary artefact
| A Rust compiler (`oikos-syntax`, `oikos-parser`, `oikos-desugar`, `oikos-check`).
| A long-running service (AffineScript port at `bot-integration-affine/`, in flight; previous ReScript implementation retired 2026-05-28) + a Haskell analyser.

| Domain
| Macroeconomics: stocks, flows, sectors, Godley matrices, fiscal periods.
| Software engineering: carbon score, Pareto distance, complexity, coupling.

| Lives at
| Repo root: `Cargo.toml`, `oikos-*/` crates, `spec/`, `README.adoc`.
| `bot-integration-affine/`, plus `analyzers/`, `policy-engine/`, `ARCHITECTURE.md`.

| Roadmap
| link:../ROADMAP.adoc[`docs/ROADMAP.adoc`]
| Phase 5 scaffold landed in PR #35; follow-ups gated on AS stdlib `Json` v0.3 + http-capability-gateway externs.

| Status (2026-05)
| Scaffold; desugar blocked on Ephapax parser milestone.
| AffineScript port scaffold; cross-module type-check + JSON payload extraction + HTTP-server accept loop pending upstream stdlib gates; *not deployed*; domain not registered.

| Licence
| EUPL-1.2
| MPL-2.0

| GitHub identifier
| https://github.com/hyperpolymath/oikos[repo `hyperpolymath/oikos`]
| https://github.com/apps/oikosbot[App `oikosbot`] (`slug: oikosbot`, id `2538504`)
|===

== Why the shared name

οἶκος is the Greek root of both "ecology" and "economy".  The DSL uses it
because it models the *economy* (macroeconomic accounting).  The bot uses it
because it does *eco* (ecological) + *econ* (economic) analysis of code.
The conceptual link is real, but the products do not share code, schemas, or
deployment surface.

== How to refer to them

[cols="1,2,2"]
|===
| Use | Don't use | Why

| "the Oikos DSL" or just "Oikos"
| "Oikos Bot" when you mean the DSL
| The DSL was here first and is what most of the repo is about.

| "OikosBot" or "the bot"
| "Oikos" when you mean the App
| Avoid silent demotion — "Oikos" alone means the DSL.

| `bot-integration-affine/...` paths
| `oikos/...` when you mean bot files
| File paths disambiguate without prose. (Pre-2026-05-28 the bot lived at `bot-integration/`; see git history.)
|===

== If you came here because you were confused

Look at where you are in the repo:

* In `README.adoc`, `Cargo.toml`, `oikos-syntax/`, `oikos-parser/`,
  `oikos-desugar/`, `oikos-check/`, `spec/`, `docs/DESIGN.adoc`,
  `docs/ROADMAP.adoc` → you're in the *DSL*.
* In `bot-integration-affine/`, `analyzers/`, `policy-engine/`,
  `ARCHITECTURE.md`, `DEPLOY.md`, `docs/GITHUB_APP_SETUP.md` →
  you're in *OikosBot*.

If a file straddles both (it shouldn't, but occasionally does), file an
issue so we can split it.

== Open question

Whether to extract OikosBot into its own repository
(`hyperpolymath/oikosbot`) is undecided.  Arguments either way are recorded
in this doc and tracked separately.

* *Extract*: cleaner identity, separate release cycles, separate CI, separate
  issue tracker.  Cost: cross-repo refactor; redirects for inbound links;
  loss of monorepo-style atomic changes.
* *Keep together*: shared `prompts/`, shared eco/econ vocabulary, the DSL
  may eventually inform the bot's economic-analysis backend.  Cost: ongoing
  disambiguation cost (this document).

No deadline.  Decide when the cost of disambiguation exceeds the cost of
extraction.
