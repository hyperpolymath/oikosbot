This is a *navigation and provenance* document.  Per
link:DESIGN.adoc#DR-011[DR-011], it maps each Oikos collapse operation to
the structured-loss (*echo*) residue it produces, and to the
*already-proven* theorem that backs that residue's guarantee.

[IMPORTANT]
====
This document asserts *no new proof*.  Every guarantee named below is
discharged upstream — in `echo-types` (Agda, `--safe --without-K`, zero
postulates) and, for Oikos's actual desugar target, in Ephapax's
`formal/Echo.v` (Coq, 24 `Qed`, zero `Axiom`).  Oikos *inherits* these; it
does not re-prove them.  The surface syntax and desugaring that would let an
Oikos model *use* a residue are deferred (see <<status>>).
====

== The journey in one line

[source,text]
----
echo-types          Agda source of truth:  Echo f y := Σ (x : A), (f x ≡ y)
   │                 the unary/decoration shadow is `LEcho` (linear ⊑ affine)
   ▼
EchoTypes.jl        executable finite-domain shadow (falsifies by counterexample)
   │
   ▼
Ephapax             formal/Echo.v — Coq port of EchoLinear + EchoResidue
formal/Echo.v       (24 Qed, 0 Admitted, 0 Axiom)   ◀── Oikos's desugar target
   │
   ▼
Oikos (this repo)   provenance-decorated transfer / close / convert
                    desugars the affine echo residue into Ephapax L3
----

The cross-repo pin lives at `echo-types/proofs/agda/EchoEphapaxBridge.agda`
(`ephapax-L3-weaken`, `ephapax-L3-no-section-collapse`).

== Operation → residue → backing theorem

Each Oikos operation is an *irreversible collapse*.  Bare linear typing
makes the collapse safe (no double-spend) but *forgetful*.  The affine echo
residue is the retained, discardable witness of what was forgotten.

[cols="1a,2a,2a,2a", options="header"]
|===
| Oikos collapse | What is forgotten | Echo residue | Backing theorem (Agda / Coq)

| `transfer A → B`
| which source flow funded the value now at `B`
| affine echo on the linearly-consumed source value
| `EchoLinear.weaken` / `weaken`

| `close from P into Q`
| the intra-period decomposition behind the closing balance
| affine echo on the region transfer
| `EchoLinear.affine-canonical` / `affine_canonical`

| `convert` (FX, dated rate)
| the pre-conversion currency identity of the amount
| affine echo on the source `Money[C]`
| `EchoLinear.weaken-collapses-distinction` / `weaken_collapses_distinction`

| Godley column netting
| the gross `+`/`−` components behind a net-zero column
| read-only residue over the column's signed entries
| `EchoResidue.no-section-collapse-to-residue` / `no_section_collapse_to_residue`
|===

== Why bare linear is not enough (the load-bearing negative)

The temptation is to record provenance in a runtime side-table after the
collapse.  The estate already has the theorem that says this is unsound:

* `EchoLinear.no-section-weaken` — there is *no* left inverse to `weaken`.
  Once a value is degraded to its affine residue, no total function recovers
  the original witness.  In Oikos terms: *you cannot reconstruct which flows
  composed a balance from the balance alone.*
* `EchoLLEncoding.ll-encoding-gap` + `gap-paired` — the standard linear-logic
  shadow encoding (`!A := 1`) admits a section, i.e. it is *too weak* to be a
  faithful residue.  A residue that can be trivially rebuilt records nothing.

Together these say: provenance must be carried *at the type level, from the
point of collapse*.  That is precisely what the affine echo residue does, and
why DR-011 admits it rather than leaving provenance to runtime.
