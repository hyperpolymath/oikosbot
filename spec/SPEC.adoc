== Introduction

Oikos is a domain-specific language for *stock-flow consistent* (SFC)
macroeconomic modelling.
It provides a typed surface language that desugars into
https://github.com/hyperpolymath/ephapax[Ephapax], inheriting Ephapax's
linear types, Tofte–Talpin regions, and typestate.

The central guarantee of Oikos is:

[quote]
____
A model that violates any accounting identity *does not compile*.
____

Oikos follows the notation of Godley & Lavoie (2007, 2012) as closely
as possible.
The primary audience is economists, not software engineers.
Keywords and identifiers use economic vocabulary throughout.

=== Design goals

1. *Accounting identity as a type invariant* — the Godley column-sum rule is
   enforced at compile time, not at test time.
2. *Dimensional safety* — monetary values carry their currency as a
   phantom type parameter; mixing GBP and USD is a compile error.
3. *Linearity* — money transferred from account A to account B is consumed
   from A at the type level, making double-counting impossible.
4. *Temporal discipline* — values are scoped to fiscal periods; crossing a
   period boundary requires an explicit `close` operation.
5. *Instrument lifecycle* — financial contracts progress through well-typed
   states; invalid transitions are rejected at compile time.

=== Relationship to Ephapax

Oikos is *not* a new language.
It is a grammar and desugaring pass over Ephapax.
The semantic guarantees come entirely from the Ephapax type system:

|===
| Oikos construct | Ephapax mechanism
| `Stock GBP` | Linear value with phantom type `Gbp`
| `Flow GBP` | Linear value with phantom type `Flow<Gbp>`
| Fiscal period | Tofte–Talpin region
| `transfer A → B` | Linear consume/produce pair
| Godley matrix | Module-level linear type constraint
| Instrument typestate | Ephapax typestate machine
|===

== Lexical structure

=== Character set

Oikos source files are UTF-8.
Unicode arrows (`→`, U+2192) are permitted and preferred;
the ASCII digraph `\->` is accepted as a fallback.

=== Comments

Line comments begin with `--` and extend to end of line.
Block comments are not supported in this version.

[source,oikos]
----
-- This is a line comment.
account wages : Flow GBP   -- Inline comment.
----

=== Keywords

----
model    period   rate      account  sector    instrument
godley   transfer convert   close    from      into
via      on       states    initial  transitions
asset    liability  amount  description  balance  fraction
Stock    Flow      FiscalYear  HalfYear  Quarter  Month  Week
Draft    Issued   PartiallyPaid  Settled  Void
----

=== Identifiers

An identifier begins with a Unicode letter or underscore and continues with
letters, digits, or underscores.
Examples: `deposits`, `FY2025`, `GBP_USD_2025Q4`, `Households`.

=== Numeric literals

Decimal numerals may use underscores as digit separators.
No hex, octal, or binary literals are defined (they have no economic meaning).

----
2500.00        -- plain decimal
2_500.00       -- underscores for readability (preferred)
1_234_567.89   -- millions separator
----

=== Currency codes

A currency code is an ISO 4217 three-letter alphabetic code written as an
identifier immediately following a numeric literal:

----
2_500.00 GBP
1_000.00 USD
----

=== Dates

Dates are written in ISO 8601 format `YYYY-MM-DD`:

----
2025-04-01
2026-03-31
----

== Type system overview

=== Stocks and flows

A **stock** is a quantity measured at a point in time.
A **flow** is a quantity measured over a period.

[source,oikos]
----
account deposits : Stock GBP   -- balance sheet item
account wages    : Flow  GBP   -- income statement item
----

These are *distinct types* at the Ephapax IR level.
The expression `deposits + wages` is a type error because you cannot add
a stock to a flow without an explicit integration step.

=== Monetary dimensions

Every monetary value carries a currency as a phantom type parameter.
Arithmetic across different currencies is a type error.

[source,oikos]
----
account deposits_gbp : Stock GBP
account deposits_usd : Stock USD

-- The following would be a type error:
-- transfer deposits_gbp → deposits_usd { amount: 1_000.00 GBP }

-- Correct: use an explicit FX conversion
rate GBP_USD_Q4 : FxRate(GBP → USD) = 1.2650 on 2025-12-31
convert 1_000.00 GBP via GBP_USD_Q4 into deposits_usd
----

=== Fiscal period regions

Fiscal periods are Tofte–Talpin regions.
Values created within a period cannot be used outside it without an explicit
`close` operation that transfers ownership to the new period's scope.

[source,oikos]
----
period FY2024 : FiscalYear from 2024-04-01 to 2025-03-31
period FY2025 : FiscalYear from 2025-04-01 to 2026-03-31

-- Carry the end-of-FY2024 deposits balance into FY2025:
close deposits from FY2024 into FY2025
----

Omitting the `close` and then referencing `deposits` in FY2025 expressions
is a type error.

=== Godley matrix — global accounting identity

The Godley matrix is the central invariant of every SFC model.
Every column must sum to zero.
Models where any sector can end a period with non-zero net financial wealth
created ex nihilo *do not compile*.

[source,oikos]
----
godley {
    | Account          | Households | CommBanks | CentralBank |
    | deposits         | +          | -         |             |
    | advances         |            | +         | -           |
    | government_bills | +          |           | -           |
}
----

Column interpretation: `+` means the sector's balance in that account
increases; `-` means it decreases; an empty cell means the sector does
not participate.

Checking rule: for every column, the count of `+` entries must equal the
count of `-` entries.
(Actual-amount balancing is enforced in the Ephapax type checker once the
desugaring pass is wired in.)

=== Financial instrument typestate

Financial instruments progress through a lifecycle of states.
Transitions not declared in the instrument's typestate machine are
compile errors.

[source,oikos]
----
instrument Invoice : GBP {
    states      Draft, Issued, PartiallyPaid, Settled, Void
    initial     Draft
    transitions {
        Draft          → Issued
        Draft          → Void
        Issued         → PartiallyPaid
        Issued         → Void
        PartiallyPaid  → Settled
        PartiallyPaid  → Void
    }
}
----

=== Linear money transfer

The `transfer` expression is linear: it consumes the specified amount
from the source account and produces it in the destination account.
At the Ephapax IR level the source value is consumed (cannot be used again)
and the destination value is freshly produced.

[source,oikos]
----
transfer wages → deposits {
    amount:      2_500.00 GBP
    description: "Monthly salary payment"
}
----

Double-counting — using the same monetary value as both source and
destination, or referencing a consumed account — is a type error.

== Top-level syntax

=== Model declaration

A source file contains exactly one `model` declaration.

[source,oikos]
----
model <Name> (period: <PeriodRef>) {
    <items>
}
----

`<PeriodRef>` must name a `period` declared within the model body.
It becomes the active region for all top-level transfer and close
expressions.

=== Complete example

[source,oikos]
----
-- Minimal two-sector SFC model (Godley & Lavoie, 2007, Model SIM)
model ModelSIM (period: FY2025) {

    period FY2025 : FiscalYear from 2025-01-01 to 2025-12-31

    -- Accounts
    account wages         : Flow  GBP
    account taxes         : Flow  GBP
    account consumption   : Flow  GBP
    account govt_spending : Flow  GBP
    account deposits      : Stock GBP
    account money         : Stock GBP

    -- Sectors
    sector Households {
        asset     deposits
        liability  -- (none in this minimal model)
    }

    sector Government {
        asset     money
        liability deposits
    }

    -- Godley transactions matrix
    godley {
        | Account       | Households | Government |
        | wages         | +          | -          |
        | taxes         | -          | +          |
        | consumption   | +          | -          |
        | govt_spending | +          | -          |
        | deposits      | +          | -          |
    }

    -- Period body: transfer expressions
    transfer govt_spending → wages      { amount: 100.00 GBP }
    transfer wages         → taxes      { amount: 20.00  GBP }
    transfer wages         → consumption{ amount: 80.00  GBP }
    transfer consumption   → deposits   { amount: 80.00  GBP }
}
----

== Grammar (draft BNF)

This section gives an informal BNF.
A precise Chumsky grammar will be provided in `oikos-parser/src/parser.rs`
once the syntax is stable.

----
model        ::= 'model' IDENT '(' 'period' ':' IDENT ')' '{' item* '}'

item         ::= period_decl
               | rate_decl
               | account_decl
               | instrument_decl
               | sector_decl
               | godley_block
               | transfer_expr
               | convert_expr
               | close_expr

period_decl  ::= 'period' IDENT ':' period_kind 'from' DATE 'to' DATE
period_kind  ::= 'FiscalYear' | 'HalfYear' | 'Quarter' | 'Month' | 'Week'

rate_decl    ::= 'rate' IDENT ':' 'FxRate' '(' IDENT '→' IDENT ')'
                 '=' NUMBER 'on' DATE

account_decl ::= 'account' IDENT ':' account_kind IDENT
account_kind ::= 'Stock' | 'Flow'

instrument_decl ::= 'instrument' IDENT ':' IDENT '{'
                      'states'      state_list
                      'initial'     IDENT
                      'transitions' '{' transition* '}'
                    '}'
state_list   ::= IDENT (',' IDENT)*
transition   ::= IDENT '→' IDENT

sector_decl  ::= 'sector' IDENT '{' sector_item* '}'
sector_item  ::= 'asset' IDENT | 'liability' IDENT | account_decl

godley_block ::= 'godley' '{' godley_table '}'
godley_table ::= godley_header godley_row*
godley_header::= '|' 'Account' ('|' IDENT)+ '|'
godley_row   ::= '|' IDENT ('|' godley_sign)+ '|'
godley_sign  ::= '+' | '-' | /* empty */

transfer_expr ::= 'transfer' account_ref '→' account_ref '{' transfer_body '}'
transfer_body ::= 'amount' ':' money_expr ( 'description' ':' STRING )?

convert_expr ::= 'convert' money_expr 'via' IDENT 'into' account_ref

close_expr   ::= 'close' account_ref 'from' IDENT 'into' IDENT

money_expr   ::= NUMBER IDENT                           -- literal
               | 'balance' '(' account_ref ')'          -- full balance
               | 'fraction' '(' NUMBER ',' account_ref ')' -- fraction

account_ref  ::= IDENT

NUMBER       ::= [0-9][0-9_]* ('.' [0-9][0-9_]*)?
IDENT        ::= [A-Za-z_][A-Za-z0-9_]*
DATE         ::= [0-9]{4} '-' [0-9]{2} '-' [0-9]{2}
STRING       ::= '"' [^"]* '"'
----
