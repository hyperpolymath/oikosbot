== Data Layer — Single Store (VeriSimDB)

*Status:* in-progress (design/config done; runtime deferred)

The data layer is retargeted from the twin ArangoDB (graph/document) + Virtuoso
(RDF/semantic) design to a single
https://github.com/hyperpolymath/verisimdb[VeriSimDB] identity-consonance store.
One identity is an octad of modal witnesses; the graph + document witnesses
subsume ArangoDB and the semantic witness subsumes Virtuoso/RDF (see
link:ARCHITECTURE.adoc[ARCHITECTURE.adoc] and
link:.machine_readable/6a2/META.a2ml[META.a2ml] ADR-001).

=== Done

* [x] Retire ArangoDB + Virtuoso assets (recoverable in git)
* [x] Keep the eco/econ ontology as the semantic-witness seed at `databases/ontology/eco.ttl`
* [x] Single `databases.verisimdb` block in `config/oikos.yaml` (REST endpoint + octad map)
* [x] Retarget the policy-engine knowledge-graph bindings (SPARQL/Virtuoso → VCL/VeriSimDB; comments only)
* [x] Integration descriptor `.machine_readable/integrations/verisimdb.a2ml`
* [x] ADR-001 recorded in `META.a2ml`
* [x] Client design + AS stub scaffold (`docs/verisimdb-client.adoc`, `bot-integration-affine/src/VeriSimClient.affine`)

=== Remaining (deferred / gated)

* [ ] Implement the client — replace the stub externs (`VeriSimClient.affine`) with a real Zig-FFI adapter (mirror `hpm-*`) or a REST impl, per `connectors/shared/openapi/verisim-api-v1.yaml`
* [ ] Verify endpoint coverage (octad CRUD + search + drift) at wire-up time
* [ ] Persist analysis results + code-graph + build metrics into the octad witnesses
* [ ] Gated on: VeriSimDB maturity (v0.1.0) and the AffineScript transport (affinescript #421/#425 integration)
